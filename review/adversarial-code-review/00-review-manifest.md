## Configuration and data used

- All adversarial content was synthetic.
- Historical migration fixtures were generated by tagged `v1.4.1` and `v2.0.0` source in detached worktrees, then migrated by current `main`.
- The redirect probe used loopback servers and a synthetic `Authorization` value. No provider request or credential was used.
- Public GitHub checks read ref/tree metadata only; restricted font binaries were not downloaded or copied into the review artifacts.

## Areas not exercised

- Real MLX model loading, long generation, embeddings, model switching under load, and XPC memory pressure
- Real CourtListener and other credentialed connector calls
- Thread Sanitizer, Address Sanitizer, Instruments, and long-running resource profiling
- Disk-full fault injection, real read-only volumes, corrupt production-size databases, and multi-process launch races
- OCR quality across languages and large scanned matters
- Attorney-domain validation of legal ranking, jurisdiction, quotation, and Bluebook outcomes
- Notarization submission or release publication from reviewed `main`
- Signed Release runtime smoke test of model load plus generation
